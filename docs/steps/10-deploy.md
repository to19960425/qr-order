# Step 10: デプロイ

Vercel へのデプロイと本番環境の設定を行う。

## ゴール

- アプリが Vercel 上で動作する
- 本番環境の環境変数が正しく設定されている
- QRコードが正しいURLで生成される

---

## タスク

### 10.1 Vercel プロジェクト作成

- [ ] Vercel にGitHubリポジトリを接続
- [ ] フレームワーク: Next.js を選択
- [ ] ビルドコマンド: `npm run build`

### 10.2 環境変数の設定

- [ ] Vercel の Environment Variables に設定:
  - `NEXT_PUBLIC_SUPABASE_URL`
  - `NEXT_PUBLIC_SUPABASE_ANON_KEY`
  - `NEXT_PUBLIC_APP_URL`（Vercelのデプロイ先URL）

### 10.3 ビルド確認

- [ ] `npm run build` がローカルで成功する
- [ ] Vercel でのデプロイが成功する

### 10.4 本番動作確認

- [ ] お客様側: QRコード読み取り → メニュー表示 → カート → 注文確定 → 完了画面
- [ ] 管理側: ログイン → 注文ダッシュボード → リアルタイム更新 → 完了操作
- [ ] 管理側: メニュー管理（CRUD + 画像アップロード）
- [ ] 管理側: 席管理（CRUD + 開閉 + QR + PDF）
- [ ] テーブルクローズ中のQR読み取りで営業時間外画面が表示される

### 10.5 QRコードの確認

- [ ] 生成されるQRコードのURLが `NEXT_PUBLIC_APP_URL` を使用していることを確認
- [ ] 実際にQRコードを読み取ってアクセスできることを確認

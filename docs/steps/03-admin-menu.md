# Step 03: 管理画面 - メニュー管理

メニューアイテムとカテゴリのCRUD、画像アップロード、並び替え機能を実装する。

## ゴール

- カテゴリの追加・編集・削除ができる
- メニューアイテムの追加・編集・削除ができる
- メニュー画像をアップロードできる
- カテゴリ・メニューアイテムの並び替えができる

---

## タスク

### 3.1 メニュー管理ページの骨格

- [ ] `src/app/admin/menu/page.tsx` を実装
  - Server Component でカテゴリ + メニューアイテムを取得
  - Client Component にデータを渡す構成
  - カテゴリごとにセクション分けして表示

### 3.2 カテゴリ管理

- [ ] カテゴリ一覧表示
- [ ] カテゴリ追加機能（インライン入力 or モーダル）
- [ ] カテゴリ名の編集
- [ ] カテゴリ削除（確認ダイアログ付き、「この操作は取り消せません。配下のメニューも削除されます」）
- [ ] カテゴリの並び替え（↑↓ ボタン、`sort_order` を更新）

### 3.3 メニューアイテム一覧表示

- [ ] カテゴリ別にメニューアイテムをリスト表示
  - 画像サムネイル、名前、価格、説明（抜粋）を表示
- [ ] 「メニューを追加」ボタン（各カテゴリセクション内）

### 3.4 メニューアイテム追加・編集モーダル

- [ ] shadcn/ui の Dialog コンポーネントを追加
- [ ] フォームフィールド:
  - 名前（必須）
  - 説明（任意）
  - 価格（必須、整数）
  - カテゴリ選択（必須）
  - 画像アップロード
- [x] バリデーション:
  - 名前: 空文字チェック
  - 価格: 正の整数チェック
- [ ] 追加時: `menu_items` に INSERT
- [ ] 編集時: `menu_items` を UPDATE

### 3.5 画像アップロード

- [ ] ファイル選択UI（プレビュー付き）
- [x] クライアント側バリデーション:
  - ファイルサイズ: 5MB以下
  - ファイル形式: JPEG, PNG, WebP
- [ ] Supabase Storage（`menu-images` バケット）にアップロード
  - ファイル名: `{uuid}.{拡張子}`
- [ ] アップロード後のURLを `menu_items.image_url` に保存
- [ ] 編集時: 既存画像のプレビュー表示、新しい画像で上書き可能

### 3.6 メニューアイテム削除

- [ ] 確認ダイアログ付き
- [ ] `menu_items` から DELETE
- [ ] Storage の画像も削除（画像がある場合）

### 3.7 メニューアイテムの並び替え

- [x] 並び替えロジック実装済み（`src/lib/sort-order.ts` + テスト）
- [ ] 各メニューアイテムに ↑↓ ボタン（UI）
- [ ] `sort_order` を入れ替えて UPDATE（DB連携）
- [ ] 先頭アイテムの ↑ と末尾アイテムの ↓ は disabled

### 3.8 動作確認

- [ ] カテゴリのCRUDが正常に動作する
- [ ] メニューアイテムのCRUDが正常に動作する
- [ ] 画像のアップロード・表示が正常に動作する
- [ ] 並び替えが正常に動作する
- [ ] カテゴリ削除時に配下のメニューアイテムも削除される

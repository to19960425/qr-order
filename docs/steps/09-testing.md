# Step 09: テスト

単体テスト（Vitest）とE2Eテスト（Playwright）を実装する。

## ゴール

- カートロジック・ユーティリティ関数の単体テストが通る
- コンポーネントの基本表示テストが通る
- 注文フロー・管理画面フローのE2Eテストが通る

---

## タスク

### 9.1 単体テスト: useCart フック

- [ ] `src/hooks/__tests__/useCart.test.ts` を作成
  - カートへのアイテム追加
  - アイテムの削除
  - 数量変更（増加・減少・0で削除）
  - 合計点数の計算
  - 合計金額の計算
  - localStorage への永続化・復元
  - カートのクリア
  - テーブルトークンごとのカート分離

### 9.2 単体テスト: ユーティリティ関数

- [ ] `src/lib/__tests__/utils.test.ts` を作成
  - 価格フォーマット関数のテスト（`1850` → `¥1,850` 等）
  - エッジケース（0、大きな数値）

### 9.3 単体テスト: コンポーネント

- [ ] コンポーネントの基本表示テスト
  - MenuCard: 名前・価格・画像が表示される
  - CartItem: 数量・価格が表示される
  - FloatingCartBar: 点数・金額が表示される
  - OrderCard: 注文番号・テーブル番号・注文内容が表示される

### 9.4 E2Eテスト: 注文フロー

- [ ] `e2e/order-flow.spec.ts` を作成
  - メニュー画面にアクセスし、メニューが表示される
  - カテゴリタブで絞り込みができる
  - メニューをカートに追加できる
  - カート画面で数量変更ができる
  - 注文を確定し、完了画面が表示される
  - 追加注文でメニュー画面に戻れる

### 9.5 E2Eテスト: 管理画面フロー

- [ ] `e2e/admin-flow.spec.ts` を作成
  - ログイン画面でログインできる
  - 注文ダッシュボードが表示される
  - 注文を完了にできる
  - メニュー管理でメニューの追加・編集・削除ができる
  - 席管理でテーブルの追加・開閉・QR生成ができる

### 9.6 動作確認

- [ ] `npm run test`（Vitest）で全テストが通る
- [ ] `npx playwright test` で全E2Eテストが通る
